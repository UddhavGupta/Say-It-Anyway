Core concept:
Users join a shared digital prompt-card session. Multiple people should be able to access and play the same session at the same time from laptops, phones, and tablets. Everyone in the same room sees the same active mode, card, level or filter, deck order, and room state in real time.

Product architecture:
The app has three game modes:

1. Classic
Description: The default general-purpose conversation game. Good for acquaintances, friends, dates, and small groups.
Includes three levels:
- Read the Room: low-pressure, playful, observant, funny, easy-entry prompts.
- Beneath the Surface: deeper personal stories, identity, ambition, family, friendship, values, patterns, and self-awareness.
- Say It Anyway: vulnerable, direct, emotionally honest, reflective prompts.

2. The Long Game
Description: Advanced prompts for people who already have real context with each other. This is for long-term couples, close friends, and people who have been on enough dates that basic get-to-know-you questions feel boring.
Includes 200 cards.
This mode is not adult by default. It should be mature and relationship-oriented, but still safe for non-sexual conversations.
The Long Game should have optional filters:
- All
- Couples
- Close Friends
- Dating

3. After Dark
Description: Hidden adults-only spicy mode.
Includes 200 cards.
This mode should be hidden by default and should not appear in the visible mode selector until unlocked.
After Dark content is for consenting adults. It should remain tasteful, modern, and emotionally intelligent. Do not make it crude, pornographic, coercive, exploitative, or graphic.

Room/session logic:
Support two ways to join:

1. Fixed Code Shared Room
There should be a fixed default room code, for example “SAYIT”. If a user enters this fixed code, they join the same canonical shared room. Anyone using this fixed code should see the same synced game state.

2. Create New Session
Users can click “Create Room” to generate a unique 6-character alphanumeric room code. Exclude confusing characters like O, 0, I, and 1. Show the generated room code clearly with a “Copy Code” button. Other players can join by entering that code.

Before joining a room:
- Ask the user for a display name.
- Store display name locally so they do not have to re-enter it every time.
- Do not require login for V1.
- Do not require payment for V1.

Card filtering:
- Classic mode filters by level.
- The Long Game filters by relationship_context: all, couples, close_friends, dating.
- After Dark only appears after after_dark_unlocked is true.
- Hidden and adult cards should never appear in Classic or The Long Game.

Deck behavior:
- Cards should be shuffled within each selected mode and level/filter per room.
- The shuffled order should persist for that room.
- If a player refreshes or rejoins, they should see the current shared card.
- If the deck reaches the end, show “You’ve reached the end of this deck” and offer to reshuffle.
- Include Next Card, Previous Card, Shuffle Deck, Reset Room, Skip Card, and Copy Room Code controls.
- Skip Card should advance the card without judgment. For After Dark, show a subtle reminder: “Skip freely. No explanations needed.”

After Dark unlock behavior:
After Dark should be intentionally hidden.
Allow unlocking through a subtle Settings control and/or by typing the unlock phrase “AFTERDARK”.
When a user attempts to unlock After Dark, show an adults-only confirmation modal:
“After Dark includes spicy prompts intended for consenting adults. Play respectfully. Anyone can skip any card, anytime.”
Buttons:
- Unlock After Dark
- Cancel

Once unlocked:
- Add After Dark to the mode selector for everyone in that room.
- Store after_dark_unlocked per room.
- Add a “Lock After Dark” option in room settings.
- If the room is reset, After Dark can stay unlocked unless the user explicitly chooses Lock After Dark.
